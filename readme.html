What is a RAM? How does it work?

A RAM is a model machine that enables computation through a set of operators and registers. Registers are in-/direct adressable and serve as the memory of the machine. It has basic arithmetic functions and a finite set of more complex instructions. A RAM is capable of computing any computable sequence and thus is (in it's extended RASP-form) equivalent of the universal Turing machine. For further information see:Wikipedia - RAM

How to use?

Please follow these steps to set the simulator up:

  • write your program using the instruction set
  • (optional) define allocation of the initial registers. Plus adds an additional input field. X deletes all input fields
  • A click on "ok" ends the editmode and makes the program runnable
  • A click on "edit" sets the program in editmode and resets all execution parameters

Please follow these steps to control the execution:

  • the controls are only usable if the program is not in editmode
  • "next" ,makes the RAM execute the next command
  • "play" ,calls "next" every second
  • "pause" ,pauses the automated execution
  • "reload" ,resets all execution parameters

Instruction Set First steps

  • RA = B ,where A is the registeradress and B can be of one of the following types: integer value, registeradress, indirect registeradress
  • RA = (RB), where A is the registeradress and (RB) is the indirect adressment of the adress saved in register b.
  • RA = B op C ,op has an arithmetic operator (+,-,*,/). The division operator is integerbased
  • GOTO ZL ,continues the execution of the program on line L
  • GGZ B ZL ,continues the execution of the program on line L, in case B (integer value, registeradress, indirect registeradress) is greater then 0
  • GLZ B ZL ,continues the execution of the program on line L, in case B (integer value, registeradress, indirect registeradress) is less then 0
  • GZ B ZL ,continues the execution of the program on line L, in case B (integer value, registeradress, indirect registeradress) is equal to 0
  • HALT ,stops the RAM (this command is mandatory and has to be once at the end of each program)

Ui Elements First steps

This RAM has two modes.

  • Editormode: Enter your program into the editing field and confirm with OK. You can add registers by pressing "add". You may give them a default value used at initialisation of the RAM.
  • Executionmode: Push "Play" to start execution or use "Next" to only do one step at a time. "Reload" reloads the entered program and resets the registers to their defaults set in editmode.

During execution

  • the last executed line of the program is marked in the program table
  • the accumulator is getting updated during execution
  • the costs are getting updated during execution

Sample Program

R1 = 10
R10 = 20
R2 = (R1) //indirectly adresses r10
R2 = R2 - 1
GGZ R2 Z4 //loops 20x then stops
HALT
